## 准备
- [ ] 邮箱登录（= 内嵌钱包 `0x5589`，已预存 10000 mUSD + INJ）
- [ ] 备好一个**小 .zip**（任意文件压一下）+ 一个**能打开的链接**（`https://example.com` 也行）

## A. 一次过（happy path）
- [ ] 立新承诺 → **🤝 交付托管** → 填开发需求 + 验收标准，**修改次数=2**，金额 50，天数 7 → 托管创建
- [ ] 期望：跳详情页，状态「等待交付」，有**复制交付邀请链接**
- [ ] 复制链接 → 同一手机新标签打开 `/d/<id>#<secret>` → 「接受委托」
- [ ] 期望：状态「等待交付」，出现上传区
- [ ] 选 .zip + 填 demo 链接 → 提交交付（签名一次 + 两笔交易）
- [ ] 期望：状态「验收中」
- [ ] 回详情页 `/pact/<id>` →「**打开 demo 实测**」能跳转 → 点「✅ 满意，放款」
- [ ] 期望：状态「已结算」，出现「📁 下载源码」→ 点它（再签名一次）能下回 .zip ✅
- [ ] **反验证**：放款前访问 `/api/deliver/<id>/source` 应被拒（源码只有放款后给委托人）

## B. 修改环（治无限改稿）
- [ ] 同上创建（修改次数=2）→ 接受 → 交付 → 验收中
- [ ] 委托人点「提修改」→ 写一句问题 → 提交（签名 + 交易）
- [ ] 期望：状态「修改中」；修改次数显示 **1/2 已用**
- [ ] 交付方页能看到**委托人留言** + 一行 AI「是否 in-spec」参考
- [ ] 交付方重新上传 + 提交 → 回「验收中」→ 委托人点「放款」→ 已结算 ✅

## C. 终局裁决（最快：改次数=0）
- [ ] 立新承诺，**修改次数=0** → 接受 → 交付 → 验收中
- [ ] 委托人显示的是「⚖️ 裁决」按钮 → 点它
- [ ] 期望：状态「终局裁决中」→ 十几秒后自动变「已结算」（放款或退款，看 AI 判）
- [ ] （Judge 后台收到 `ArbitrationRequested` → AI 旗舰对照验收标准判 → 上链）

## D. 回归（确认重部署没搞坏旧功能）
- [ ] 立新承诺 → **🥷 自律打卡** → 聊两句生成方案 → 创建 → 拍照打卡 → AI 出裁决 ✅
- [ ] 自律承诺勾选「邀请见证人」→ 复制见证链接打开 → 看到打卡照片 + 能质疑 ✅

## ⏳ 暂不便手机测（窗口太长）
- 评审超时自动放款（2 天）、修改超时自动退款（3 天）、无人接单退款 —— 靠 keeper 定时触发。
  需要快测时可临时把窗口改成几分钟、单独部一个「测试版」合约专门验超时。

## 🆘 踩坑预案
- **卡在「上链中」**：Privy 偶发客户端卡顿 → 刷新重试（链是好的）
- **上传失败 / 413**：.zip 太大（>~8MB）→ 换小文件（express 10MB + nginx 12MB 限额，以后调）
- **「gas limit must not be zero」**：某笔会 revert（权限/相位不对）→ 核对当前状态是否匹配操作
- **裁决迟迟不结算**：Judge 余额只剩 ~1 INJ，测多了可能没气 → 给 `0xfBA77…` 补 INJ（https://testnet.faucet.injective.network/）
