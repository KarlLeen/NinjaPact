# Ninja Pact(兑)
### 给一切小额承诺提供执行层 — The Execution Layer for Small Promises

> 把「如果 X,就 Y」变成一个可执行的对象:钱 + 自然语言条件 + AI 预言机 + 链上托管。

---

## 问题:人类 99% 的承诺,没有执行层

社会只给两种承诺配了牙齿:**大额承诺归法律**(但诉讼成本让万元以下的纠纷不值得打官司),**平台内承诺归平台**(淘宝管你和卖家,出了平台就管不着)。

剩下的呢?立给自己的 flag、说好到场的饭局、朋友间 50 块的代办、答应粉丝的周更——**小额、非正式、跨平台的承诺,执行机制为零**。于是「鸽了」成了一个轻松的词,违约的社会成本被默认为零。

不是没人想管,是管不起:人工裁决一单纠纷的成本,远高于 200 块的标的。

## Why Now:裁决成本在 2025–26 年第一次跌穿了费率线

- **AI 裁决边际成本归零**:DeepSeek V4 级模型把一次多模态证据验证打到几分钱——两年前用 GPT-4V 做同样的事,费率倒挂,商业上不成立
- **执行成本归零**:稳定币 + Injective 近零 gas,托管和结算不再需要任何机构
- **入口成熟**:agent 钱包与账户抽象,让普通人第一次用得起链

三条线同年交汇。这个产品在 2024 年之前造不出来,在 2027 年之后轮不到我们造。

## 第一个场景:带牙齿的 flag(自律质押)

立 flag 时锁入一笔真钱 → **AI 监工**每日验收证据(可信拍摄 + 多信号交叉)→ 成功,本金即刻全退,获得 soulbound 守诺勋章与履约信用;失败,本金进入资金池锁定 6 个月后**原路全额退还**。

**没有人损失本金。** 这不是赌博,是给承诺装上流动性的代价。失败的惩罚是真实的(六个月拿不回钱),但本金永远是你的——self-improvement should help ourselves, not punish ourselves。

## 为什么这个设计聪明(两刀)

**1. 退还制同时封顶了作弊收益和 AI 错判代价。** 伪造 30 天证据骗过裁判,你「偷」到的只是自己的钱提前六个月解锁——几块钱的时间价值,低于伪造成本,攻击在经济上不成立。同理,AI 错判的代价也只是流动性,不是本金。最难做好的组件(AI 裁判)被机制设计降级为非关键组件,v1 即可安全上线。

**2. 裁判、托管、受益人三权分立。** 这个品类的在位者(StickK / Beeminder / Forfeit)验证了 18 年需求,但全都「裁判兼庄家」——判你失败,它就赚钱(Beeminder 的 FAQ 原话:收罚金就是我们的商业模式)。我们:合约管钱,AI 断案且裁决理由全部上链可审计,平台只收与输赢无关的固定费率。Web2 在位者修不了这个病——修了就杀死自己的商业模式。

## 一台引擎,三拍叙事:入口 → 生意 → 终局

**引擎** = 一组经审计的托管模板(单锁 / 对锁 / 奖池 / 里程碑 / 押金可退)+ AI 裁决协议 **Judge**(分级裁决、申诉、判例库,以 ERC-8004 注册为 Injective 上的链上 agent)。一笔交易本来就是一对互相承诺——「你交付 X,我付 Y」——所以同一台引擎,按风险阶梯逐环展开:

1. **入口:单人 flag**(零冷启动,作弊只伤自己 → Judge 的零风险练兵场)
2. **双人挑战 / 对赌**(「我赌你跑不进 4 小时」——对抗性举证免费强化裁判)
3. **生意:小额交易担保层**——每个微信 / Telegram 大群里都坐着一个收 1–5% 的人肉中间人:代购、约稿、接单、链上资产 OTC,全靠江湖信誉裸奔。AI 把单笔裁决成本打到几分钱后,**10–500 美元区间的交易第一次担保得起**(传统 Escrow 服务的费率结构碰不了这个区间)。立约时 Judge 强制双方把验收标准说清楚再锁钱——大多数 C2C 纠纷死于约定模糊,一个前置澄清的 AI 本身就是产品。标的从可验证交付的数字商品与服务起步,白名单制
4. **开放 SDK**(任何 app 一行代码接入「担保 / 承诺」能力,我们退到协议位)
5. **终局:Agent 经济的验收层**(agent 向人或另一个 agent 承诺交付时,谁来验收?Judge 就是那个 Evaluator——而 agent commerce 本质上就是交易担保的 agent 版,同一个产品,换了客户)

**先例**:支付宝从淘宝的担保交易长成支付帝国。担保是入口,结算是生意。

## 商业模式与护城河

**收入阶梯**:开局固定服务费(成本价 +,从第一单起覆盖 AI token 成本、自负盈亏)→ 浮存金收益(透明化、链上可查)→ **担保结算的 GMV 抽成(核心生意:每一笔经过引擎的交易留下费率,随交易量复利)** → 履约征信查询。

**复利资产**:判例库(普通法式积累,裁决越多越一致)、**履约信用图谱**(为法律够不着的领域建一个征信局,在交易场景里即卖家信誉)、证据集成网络(健康 app / GitHub / 物流 API,越接越宽)。

**对 Injective 的双向漏斗**:自律入口把圈外人静默带进 Injective(嵌入式钱包,登录即开户);担保层把真实结算量与稳定币沉淀留在 Injective 上。一头带人,一头带钱。
